📘 Aperçu & concepts fondamentaux

Le module de facturation des vols gère la facturation des membres, les réductions par forfaits et l’intégration comptable. Les vols sont d’abord facturés au prix brut, puis les rabais forfaitaires sont appliqués via un journal REM dédié (périodique).

✈️ 1. Facturation brute (journal FL)
Vol 1h planeur : 100€ débit 411 / crédit 7062
Pas de réduction immédiate
🎁 2. Achat d’un forfait
Pack 25h à 500€ → crédit 7066, débit 411
Réduction unitaire = 80€/h
🔄 3. Ajustement REM (périodique)
Écriture REM : 6xx → 411 (montant total des réductions)
Net dû par le membre = brut − réductions
💡 Principe clé : le solde du compte 411 (créances membres) est évalué en combinant les écritures FL brutes ET l’ajustement REM en cours. Les alertes de seuil minimal se basent sur le net final.

1.2 Créer un forfait (pack)

Accès : Catalogue → Forfaits

  • Code : PACK_25H_GLIDER, Nom : Pack 25h planeur
  • Type : flight_hours (heures de vol) / winch_launches (treuils)
  • Quantité incluse : 25 (heures)
  • Compte vente : 7066 (classe 7) – peut être modifié
  • Compte charge remise : 658 (classe 6) – débité lors des réductions

1.3 Associer un forfait aux prestations (pack_applicability)

Dans la fiche du forfait, onglet Tarifs applicables : lier chaque pricing_item (ex: HEURE_CELLULE_ASK21) et indiquer le prix unitaire après remise (ex: 20€ au lieu de 100€). La réduction unitaire est automatiquement calculée (80€).

📌 Un forfait peut couvrir plusieurs types de machines (ASK21, LS8) et un même pricing_item peut être couvert par plusieurs forfaits (pack 25h, pack 50h). La logique FIFO s’applique.

1.4 Types VI (vols d'initiation)

Administration → Types VI : pour chaque type (VI, JD, STAGE), vous pouvez définir un compte de charge spécifique (sinon le compte par défaut des paramètres est utilisé).

2.2 Étapes : prévisualiser → appliquer (brouillon) → poster

  1. Filtrer les vols avec statut pending (non facturés).
  2. Développer une ligne (▼) : visualiser les lignes de tarification, les packs disponibles et le montant brut.
  3. “Appliquer (brouillon)” 📄 : crée une écriture brute dans le journal FL, enregistre les consommations de forfait (member_pack_consumptions) et met à jour l’écriture REM du pilote (brouillon).
  4. “Poster” 📤 : valide l’écriture (devient immuable). Possibilité de faire “Appliquer+Poster” directement.

2.3 Indicateurs de statut

pending en attente de facturation
applied écriture brouillon FL créée
posted écriture définitive
correcting correction en cours (vol modifié après facturation)
missing pricing version de tarif manquante → bloquant

✅ Opérations batch : sélectionner plusieurs vols → “Apply All” (transaction atomique). Si un vol échoue, tout est annulé.

🎁 3. Gestion des forfaits membres

3.1 Achat d’un forfait

Depuis la fiche membre (Membres → [membre] → Forfaits) ou via action rapide dans l’onglet Vols. L’achat déclenche :

  • Écriture comptable postée dans le journal VT : débit 411 / crédit 7066 (montant du pack).
  • Mise à jour du solde visible dans vw_member_pack_balances.
  • Si le paramètre “Recalcul après achat” est activé, les vols passés éligibles (dans la même année fiscale) sont recalculés automatiquement.

3.3 Ajustement manuel : modifier la date “Valide à partir de” (valid_from)

Lorsqu’une consommation ne doit pas s’appliquer à un vol (ex: achat postérieur hors délai de tolérance), il suffit de modifier la colonne Valid from dans la liste des consommations du membre (pencil ✏️). Si valid_from devient postérieur à la date du vol, la consommation est exclue du calcul REM. L’écriture d’ajustement REM est recalculée instantanément.
⚠️ Ce mécanisme remplace l’ancien “freeze/unfreeze”.

📅 4. Périodes REM & clôture

Les réductions forfaitaires sont regroupées par période (par défaut 30 jours). Une seule écriture de brouillon par membre et par période dans le journal REM.

  • Consulter les brouillons REM : Comptabilité → Remises périodiques – liste des membres avec total des réductions.
  • Clôturer une période : l’ensemble des écritures REM brouillon sont postées (deviennent immuables). Une nouvelle période vierge est ouverte automatiquement.
🔒 La clôture intervient généralement en fin de mois ou trimestre. Après clôture, aucune modification rétroactive des consommations n’est possible sans écriture de contre-passation.

🌐 5. Portail membre (accès externe)

Les membres peuvent consulter leurs vols, factures et solde via un portail sécurisé par jeton (lecture seule).

5.1 Activer l’accès

Membres → [membre] → Accès externe : activer l’interrupteur “Accès portail”, régénérer le jeton. Le communiquer au membre (email, impression).

5.2 Ce que le membre voit

  • Tableau de bord : solde compte 411, forfaits actifs avec quantités restantes, 5 derniers vols.
  • Liste détaillée des vols : montant brut, réduction appliquée (détail par ligne), net dû.
  • Écritures comptables le concernant (consultation).

🛠️ 7. Dépannage fréquent

Erreur "Aucune version tarifaire active" → Vérifier qu’une PricingVersion existe pour le type d’actif à la date du vol (statut actif, plage de dates).
Facturation club : "compte de charge manquant" → Configurer soit le compte de charge dans le Type VI, soit le compte par défaut dans Paramètres facturation vols.
Un forfait ne s’applique pas → Vérifier : solde restant >0, type de pack correspond à la prestation (flight_hours / winch_launches), valid_from ≤ date vol, et lien pack_applicability existe bien avec discounted_unit_price.

Vol modifié après facturation postée : le badge "correcting" apparaît. Le comptable doit créer une contre-passation de l’écriture originale, puis générer une nouvelle facturation (brouillon + poster). La traçabilité est conservée.

📖 Glossaire

411 – Compte client (créances membres)
7066 – Ventes de forfaits / packs (classe 7)
658 – Compte de charge pour remises (classe 6)
Journal FL – Écritures de vol brutes
Journal REM – Écritures d’ajustement périodique (rabais)
Valid from – Date à partir de laquelle une consommation de pack est prise en compte
FIFO – Ordre de consommation des packs : premier acheté, premier consommé
Membre club – Enregistrement sentinelle pour facturer les vols au club
Analytical asset – Machine (planeur, treuil) attachée aux lignes comptables
